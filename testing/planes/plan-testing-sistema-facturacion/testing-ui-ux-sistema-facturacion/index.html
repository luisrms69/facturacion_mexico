    Ejecución: 1. G01: Sales Invoice nuevo (pantalla inicial limpia), botón Cancel/Cancelar nativo NO VISIBLE 2. G02: Cliente y producto seleccionados, importe calculado 3. G03: Submit exitoso, documento en estado "Submitted", botón Cancel/Cancelar nativo sigue OCULTO 4. G04: Botón "Timbrar Factura" visible y activo, UI limpia sin confusión

    Invariantes UI/UX: - Cliente seleccionado correctamente de dropdown - Producto autocompletado funcionando - Cálculos automáticos funcionando - Botón fiscal aparece después de Submit - Estados documento correctos en interfaz - Botón Cancel/Cancelar nativo Frappe permanentemente oculto en FFM - Solo botones fiscales específicos visibles (evita confusión usuarios)

    Nota: Cliente con RFC real. Puede proceder con timbrado si empresa tiene RFC configurado.
